# Conditional VAE for IAM Scenarios (cVAE_iam)

This repository implements a Conditional Variational Autoencoder (CVAE) in PyTorch to model and predict integrated assessment model (IAM) time-series data using conditional text embeddings generated by the OpenAI API.

## Repository Structure

- `main.py`: The primary script for compiling the datasets, fetching text embeddings, defining the encoder/decoder architectures, and running the full training logic from scratch.
- `inference.py`: A script strictly for loading pre-trained model weights from a checkpoint to execute an inference loop over the designated test dataset, computing comprehensive performance metrics and predictions seamlessly.
- `data/`: The root dataset directory intended to hold your input trajectory Excel spreadsheets (`.xlsx`) and the underlying embedding disk caches.
- `save/`: The output directory where model checkpoints (`.pth`), variable testing JSON evaluation metrics, predictions, and loss curves are securely dumped.

## Requirements

The codebase utilizes:
- `torch`, `torchvision`, `numpy`, `pandas`, `scipy`, `matplotlib`
- `openai` (v1.x)
- `tqdm` (for console progress bars)
- `openpyxl` (Pandas required engine for `.xlsx` dependency reading)

Ensure your OpenAI API key is properly updated in the `OPENAI_API_KEY` string internally within both `main.py` and `inference.py` so the text embedding pipeline functions correctly.

## 1. Running the Training (main.py)

Before running `main.py`, confirm that the input data paths internally pointing to your data splits (`TRAINING_PATH`, `DEV_PATH`, `TEST_PATH`) accurately match your excel file locations.

**What `main.py` Does:**
1. Loads the provided `.xlsx` structured matrices sequentially extracting conditions such as `model_family`, `scenario`, `region`, and continuous target `variables`.
2. Asynchronously fetches pre-cached `text-embedding-3-large` high-dimensional OpenAI vectors mapped to your scenario description configurations as the conditioning variables for the network.
3. Automatically partitions out structural datasets via standard PyTorch DataLoaders.
4. Executes the CVAE backpropagation looping standardizing reconstruction errors (MSE) alongside a constrained Kullback-Leibler Divergence (KLD). 
5. Emits the best validated model iteration weights straight to `save/cvae_condition[MODE]/best_cvae_model.pth`.
6. Prints out standard regression benchmarks (MAE, RMSE, sMAPE, Pearson, Spearman, Category Accuracy).

To initiate training:
```bash
python main.py
```

## 2. Running Inference Only (inference.py)

The inference sequence cleanly decouples model evaluations without unnecessarily repeating training and simply uses the hyper-parameter configuration dynamically dumped gracefully from a previous `main.py` run from `training_hyperparameters.json`.

**What `inference.py` Does:**
1. Locates `best_cvae_model.pth` within your globally initialized `SAVE_DIR`.
2. Runs the defined standalone testing set (`TEST_PATH`) seamlessly utilizing the same input/embedding extraction structure.
3. Performs a complete network forward pass on the `CVAE` utilizing static evaluations.
4. Generates complex per-variable evaluation structures stored safely within `inference_metrics.json` inside your designated `SAVE_DIR`.
5. Iteratively restructures all raw numpy prediction array time series rows directly back perfectly into a completely matching pandas `DataFrame` and elegantly exports it out as an `inference_predictions.xlsx` sheet natively!

To launch predictions off a trained network run:
```bash
python inference.py
```
